Deploying blog to github.io via Hexo

前言

一直以來都很希望能夠有一個簡潔的網誌可以使用，最好是符合

  1. 簡便、簡潔
  2. 能夠發布成網頁的方式進行瀏覽
  3. 可以在任何地方寫作
  4. 寫作後的發佈能夠直觀簡單一點
    雖說Hexo + Github的作法，無法直接用iPad進行寫作與上傳，但是能夠直接用Github發布一個網頁的做法
    實在是深得我心。以前希望能夠有一個簡便的網頁，都是去下載別人設計好的html網頁，然後在伺服器上發布，並且
    買一個網域來做這件事情。後來覺得這樣很麻煩，就用常見的Wordpress託管網站的方式架站，卻又覺得過於華美、笨重。

後來發現原來Github.io可以直接發布一個靜態網頁，覺得很有趣！一個專屬你名字的網頁，似乎符合了我的需求(心情)
網路上已經有很多人寫好了如何使用Hexo + Github來建立一個自己的靜態網誌。
這邊我就留我自己的流程和中間遇到的問題

開始！

我遇到並且解決的問題

  1. hexo deploy時，輸入正確的密碼卻無法進入。| 解決方法：按官方文件所述，產生Personal access token並且將之取代為輸入的密碼，即可解決
  2. 發布後因為_config.yml中Deployment的branch設定為master，使得Github.io發布的頁面為原始產生的頁面。|解決方法：直接將branch設定為main，或者按官方文件所述，進Github.io專案中的Setting，將Page的Source改為master。
  3. 在伺服器上進行hexo deploy時，遇到了Authentication的問題

    github git-credential: “erase” operation not supported

重複的執行 hexo deploy並沒有辦法讓他跳出輸入username和password，因此參照這份文件的說明，利用下面這一行指令來重新授權hexo進行Github push
